Client Report

I designed the website with the intention of it being dynamic. I chose content and images that was minmalistic and that wouldn't overcomplicate things for my developers. My lead developer didn't reach out to me for the first two weeks. He hadn't set up the repository and hadn't organized the websites directories so the junior developers couldn't do anything. With the help of his junior developers, my lead developer figured out how to clone the repository and set up the website. The website had a slow start, but the product isn't bad. It is more or less what I expected the website to turn out. If I were to do this again I would try to get in contact with my lead developer sooner and make sure he had things under control.

Lead Developer Report

I reached out to my client the day I was assigned them. They hadn't finished their site plan so I had nothing to do on the site till a week later once the site plan was finished and shared with me. I reached out to my junior developers several times and never got a response. I originally assigned the tasks on github. After a week and I saw that none of my junior developers touched the site so I sent an email just reminding them of their assignments. After another week I sent another email asking if they have any questions and also asking them to reply to the email if they intend to do the assignment. I still haven't gotten a response out of them so I'm assuming they don't intend to work on the site, so as of now the site is only a home page. I had some technical difficulties managing the site on github since I'm still not too familiar with the site.

Junior Developer Report

I communicated to my team leads via email. There wasn't any communication issues I had as a junior developer. My assignments were clear and if I had questions the lead developer/client would answer them. I contributed a page on each of my junior developer sites. As mentioned I had some difficulties using github, but nothing that negatively effected the final product.

Reflection

Learning github was by far the most difficult part of the project. I was new to using github and it was difficult to learn how it works. I now have the github problems straightened out for this project, but still don't feel too comfortable with using github. Communication was also a struggle on some of my teams. If I were to do this again I would try to improve communication within my teams. Maybe if I had reached out early to my junior developers and confronted them in class they would have done their part of the project. To be a lead, you need to apply some pressure. I feel slightly more confident in doing real-world project, but have a long way to go before my confidence matches my performance. This final helped me learn to work in a team on a bigger project and communicate with my teammates.
